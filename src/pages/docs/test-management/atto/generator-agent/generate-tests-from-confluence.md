**Generate structured test cases directly from Confluence pages using AI.**

---

You can generate multiple test cases from Confluence pages using AI. Select the Confluence workspace and page, and Test Management by Testsigma analyzes the content to create complete, structured test cases. This article discusses generating test cases from Confluence pages.

---

> ## **Prerequisites**
> 
> Before you begin, ensure that you have a project in Test Management by Testsigma and an integrated Confluence instance with accessible pages.

---

## **Generate Test Cases**

1. From the **Dashboard**, click **AI Powered Test Case Generation** in the top-right corner.

   Alternatively, navigate to Test Cases and click **Create with AI** in the top-right corner.

2. On the **Ask AI â€“ Generate Multiple Test Cases** page, select **Confluence Pages**.

3. In the **Add Confluence Pages** dialog:
   - Select the **Confluence Space**.
   - Select one or more pages.
   - Click **Add**.

4. Click the folder icon and select the destination folder where the generated test cases should be saved.

5. Select the test case type:
   - **Documentation Template**, or
   - **Test Step Template**

6. Enter the prompt to guide test case generation, and click **Generate**.

7. Wait while Testsigma generates scenarios and the corresponding test cases.

8. Expand each scenario to review the associated test cases.

9. Select the test cases you want to include under each scenario, and click **Accept**.

10. The selected test cases are added to the chosen folder.
