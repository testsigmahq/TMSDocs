**Use Review Management to streamline test case reviews and ensure accuracy, consistency, and adherence to project standards.**

---

Review Management helps teams maintain high-quality test cases by adding a structured review workflow to Test Management by Testsigma. QA members can submit their test cases for review, and designated reviewers can approve them or return them for rework with comments. This ensures that test cases are accurate, consistent, and aligned with project standards. This article discusses Review Management in Test Management by Testsigma.

## **Enable Review Management**

1. From the left navigation bar, go to **Project > Project Settings**.

2. Turn on the **Review Management** toggle.

[[info | **NOTE**:]]
| Review Management is a project-level setting. You must enable it manually for each project where you want to use this feature.

---

## **Submit Test Cases for Review**

1. From the left navigation bar, go to **Test Cases**.

2. Open an existing test case or create a new one.

3. On the test case details page, click **Submit for Review**.

4. In the **Submit for Review** dialog, select a **Reviewer** from the dropdown menu.

5. Click **Submit for Review**.

[[info | **NOTE**:]]
| 1. You can also submit a test case for review by clicking the **Edit icon** on the test case list page and selecting **Submit for Review**.
| 2. To submit multiple test cases, select them in the list, press **Command + K**, select **Submit for Review**, and select a reviewer.

---

## **Review Test Cases**

1. On the **Dashboard**, go to **Test Case Review Status**.

2. Click any test case submitted for review. This will open the test case details page.

3. On the test case details page:
   - Click **Mark as Ready** to approve the test case.
   - Click **Mark as Rework**, add your comments in the dialog, and submit it to send it back for changes.

4. All review actions appear in the **Activity** section of the test case details page.
